Objective

Our client, Professor Dave Small, presented us with a task of creating an Artificial Intelligence (AI) for a tile-based strategy game. This game, dubbed "TigerZone" is a spinoff of the popular board game "Carcassonne". TigerZone reflects adaptations to Carassonne by including new features such as new scoring features. The full game description and rules can be found on our client's website here.

The objective of this project was to utilize agile software development skills that were discussed throughout the course. Our team utilized sprints to get specific features working and to build our overall project progress in incremental ways that helped with testing. As explained above, the goal of this project was to create an AI that could play against other teams in the class by connecting to a TCP server. A game was won by scoring points strategically by completing game features.

Sprint Outline

Sprint 1 (10/31/16 - 11/7/16)

The goal of this sprint was to begin brainstorming a solution to a variant of the game Carcassonne that is to be dubbed “Tiger Zone” by our client. Our client determined this project must have a networked competition server that has the ability for two AI clients to play two games simultaneously using TCP sockets. The two games are expected to use the same stack of tiles, with one game having Player1 start and the other game having Player2 start. The client also wants the game to have a time limit, and have a player forfeit if an illegal move is made or if time runs out.

The duration of this sprint was spent researching existing Carcassonne games, including both digital and physical variants of the game. This helped our team understand the rules and flow of the game which later was useful when completing team deliverables. In addition, the team discussed each other’s experience working with different programming languages as well as created a schedule to determine when the team would be able to meet face-to-face.

By the end of this sprint, we were able to solidfy the following (Even though these items were subject to change):

  • Project Model: Agile Methodology
  • Language: Java
  • Testing framework: JUnit
  • Testing Artifacts Through BDD Testing
  • Our Hexagonal Architecture
  • Our UML Use Cases

Sprint 2 (11/8/16 - 11/14/16)

The main objective of this sprint was to make sure that our logic for the Tile class was robust, reusable, and dynamic. We realzied that the Tile class would be the foundation for the entire game logic. As a result, we met for a couple of days to add all the necessary functions, properties, and testing methods that could be used for the Tile class. As a group, we planned to cover for special cases that a tile can have and planned for those special cases accordingly. We based the structure of the tiles to the tiles that were implemented in the game "Carcassonne."

Because our project model was utilizing agile techniques, it was important that there was an insignficant number of obstacles and that our project was moving quickly and smoothly. After the initial planning, our team formed a repository that would hold all of the source code, documentation, and testing files that we would need for our project. At the end of this sprint, our group came to the conclusion that the developers and the testers should be split. While the developers were creating the structure of the project and building the logic of the game, the testers would use the narratives from the first sprint to test against the logic of the project. Also, our team decided on the creation of GUI, which would be used for debugging and changing some of the functions easily. We planned to make a basic GUI which would not cut into the time of developing the logic of the application.

By the end of this sprint, we were able to solidfy the following (Even though these items were subject to change):

  • The Logic of the Tile
  • The Breakdown of Individuals' Duties
  • Creation of All Classes with their respective properties
